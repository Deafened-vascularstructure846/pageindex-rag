## 📘 What this app does

pageindex-rag is a Windows app for finding answers in your own files without using a vector database. It uses a reasoning-based retrieval flow to scan indexed pages, pull the most useful parts, and build a direct answer.

It works well for people who want to search local documents, notes, exports, and reports with less setup. You do not need to know how to code to use it.

## 📂 How to use it

1. Open pageindex-rag.
2. Choose the folder you want to search.
3. Let the app index the files.
4. Type a question in plain language.
5. Read the answer and check the source file links if shown.
6. Refine your question if you want a narrower result.

Example questions:

- What does this report say about cost?
- Find the main risks in these notes.
- What dates are mentioned in the project files?
- Summarize the action items from the documents.

## ⚙️ How it works

pageindex-rag uses a page-based retrieval process.

That means it:

- breaks content into smaller parts
- looks for the most relevant pages
- reasons over the matched text
- returns an answer without relying on vector search

This approach is useful when you want a simpler retrieval flow and a setup that stays close to the source text.

## 📁 Best files to index

Use files that contain clear, readable text.

Good choices:

- meeting notes
- knowledge base exports
- manuals
- SOPs
- policy documents
- research notes
- support records
- project summaries

Less useful files:

- scanned images with no text layer
- very large archives
- password-protected documents
- files with heavy formatting and little text

## 🧼 Tips for better results

- Keep your source folder focused
- Use short and clear file names
- Remove duplicate copies
- Separate personal and work files
- Re-index after you add new documents
- Ask one question at a time
- Use plain words in your search query

## 🔐 Privacy and local use

pageindex-rag is built for local document search. You control the files you index and the folders you point the app at.

If you use private files, keep them in a folder you trust and only index content you want the app to read.
